## 三、合并与处理策略

### 3.1 合并流程

由 `merge_datasets.py` 脚本自动完成：

1. **收集原始数据集**：保留 `data_realworld/collected/` 中 265 张有标注的图片
2. **收集外部数据集**：遍历每个外部数据集的 train/valid/test 目录
3. **类别映射**：按 `EXTERNAL_DATASETS` 配置表转换原始类别 ID → 目标类别 ID
4. **格式转换**：`wharf handling` 的多边形标注自动转为 bbox
5. **下采样**：对 dominant classes 执行稀疏化
6. **随机划分**：整体打乱后按 8:1:1 划分为 train/val/test
7. **重命名**：所有文件统一命名为 `img_{idx:06d}.jpg/.txt`

### 3.2 下采样策略

为解决类别极度不平衡问题（原始 helmet 类有 72,000+ 框），实施以下策略：

```python
DOWNSAMPLE_CLASSES = {"1", "4"}       # people without helmet, helmet
DOWNSAMPLE_RATIO = 0.1                # 仅保留 10%
DOWNSAMPLE_DOMINANT_THRESHOLD = 0.5   # 当 class 1/4 占样本总框数 >50% 时触发
```

**效果**：
- 合并前：约 28,777 张图片，helmet 类 72,000+ 框
- 合并后：约 5,978 张图片，helmet 类 6,450 框
- 图片数减少约 79%，但类别分布更均衡

### 3.3 数据增强策略

**不采用离线增强**（不生成新图片文件），所有增强通过 YOLOv8 训练时的**在线增强**实现：

| 增强类型 | 参数 | 说明 |
|:---|:---|:---|
| 水平翻转 | `fliplr=0.5` | 50% 概率 |
| 垂直翻转 | `flipud=0.1` | 10% 概率 |
| 随机旋转 | `degrees=15` | ±15° |
| 随机缩放 | `scale=0.5` | 50%~150% |
| 随机平移 | `translate=0.1` | 10% |
| HSV 增强 | `hsv_h/s/v` | 色调/饱和度/亮度 |
| Mosaic | `mosaic=1.0` | 四图拼接 |
| MixUp | `mixup=0.1` | 图像混合 |
| Copy-Paste | `copy_paste=0.1` | 复制粘贴 |
| Random Erasing | `erasing=0.4` | 随机擦除 |

## 六、已知问题与注意事项

1. **类别不平衡**：`helmet`（class 4）仍占训练集的 49.8%，可考虑进一步下采样或使用 Focal Loss
2. **tired 类样本少**：仅 39 个框（来自原始数据集），因外部 tired 数据集无检测标注而未合并
3. **phoning 类样本少**：仅 101 个框，建议后续补充更多打电话场景数据
4. **wharf handling 多边形转换**：由多边形取外接矩形转换而来，精度有一定损失
